# Review Seeding Guide

## วิธีการ Seed รีวิวเข้าระบบ

### ขั้นตอนการ Seed

1. **ตรวจสอบว่ามีทัวร์ในระบบแล้ว**
   ```bash
   # ถ้ายังไม่มีทัวร์ ให้ seed ทัวร์ก่อน
   npm run seed:tours
   ```

2. **รัน Seed Script**
   ```bash
   cd backend
   npm run seed:reviews
   ```

### สิ่งที่ Script จะทำ

- สร้างผู้ใช้งาน seed 25 คน (ชื่อภาษาไทย)
- สร้างรีวิวสำหรับทุกทัวร์ (10-25 รีวิวต่อทัวร์)
- สร้าง Booking ที่มีสถานะ SUCCESS สำหรับแต่ละรีวิว
- คำนวณและอัปเดตคะแนนเฉลี่ยและจำนวนรีวิวของทัวร์อัตโนมัติ
- ใช้ Transaction เพื่อความปลอดภัยของข้อมูล

### การกระจายคะแนน

- 60% รีวิว 5 ดาว
- 30% รีวิว 4 ดาว  
- 10% รีวิว 3 ดาว

### ความปลอดภัย

✅ ใช้ Transaction - ถ้าเกิด error จะ rollback ทั้งหมด
✅ ตรวจสอบว่าผู้ใช้ไม่รีวิวทัวร์เดียวกันซ้ำ
✅ สร้าง Booking ที่ถูกต้องตามกฎธุรกิจ
✅ อัปเดตคะแนนตาม AVG จริงจาก database

### หมายเหตุ

- ผู้ใช้ seed จะมี email: `seed.user1@9tours.local` ถึง `seed.user25@9tours.local`
- สามารถรัน script ซ้ำได้ แต่จะสร้างรีวิวเพิ่มเติม (ไม่ซ้ำผู้ใช้เดิม)
- ถ้าต้องการเริ่มใหม่ ให้ลบข้อมูลใน table `reviews` และ `bookings` ที่เกี่ยวข้องก่อน

### Production Deployment

Script นี้ปลอดภัยสำหรับ production เพราะ:
1. ใช้ Transaction ป้องกันข้อมูลเสียหาย
2. ไม่ลบข้อมูลเดิม
3. ตรวจสอบความถูกต้องของข้อมูลก่อน insert
4. มี error handling ครบถ้วน
